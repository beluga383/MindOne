# MindOne 贡献代理约定

## 用户可见合同

- 用户界面、帮助、状态和普通错误使用简体中文。
- 协议字段、API 路径和结构化日志键使用稳定英文标识符。
- 不得用固定响应、假余额、假证明或仅状态文件冒充真实运行状态。
- 普通错误返回 `Result`；生产代码不得使用 `panic!`、`todo!` 或 `unimplemented!`。

## 安全边界

- 禁止提交 Token、密码、OAuth Secret、数据库凭据、私钥、模型或本地配置。
- 跨主机网络使用 TLS；仅允许明确的 loopback 开发 HTTP。
- 模型只允许经结构验证的 GGUF 或 safetensors，危险反序列化格式硬拒绝。
- CLI 不直接访问数据库；账户与任务操作只通过协调服务器 API。
- 不记录 Prompt 或 Response 明文。

## 工程规则

- 金额全部使用整数 microquota，数据库结算必须处于事务中。
- 账本只追加；结果提交、结算和准备金操作必须幂等且可审计。
- 领取任务前和执行前各检查一次节点策略。
- 新行为需有确定性测试；平台能力不可用时明确降级或拒绝。
- 修改后运行相关测试，并在合并前通过 fmt、clippy 与 workspace tests。
