# ADR 0001：模型格式与结构验证边界

- 状态：已接受
- 日期：2026-07-17

## 背景

MindOne 需要同时支持 `llama.cpp` 的真实本地推理和不执行模型内任意代码的安全边界。`llama.cpp` 的可执行模型格式是 GGUF；部分其他引擎使用 safetensors。仅检查扩展名或下载来源不能证明文件安全，也不能阻止改名伪装。

## 决策

MindOne 只允许登记经过结构验证的 GGUF 和 safetensors：

- GGUF 必须具有正确 magic、受支持版本、完整且有界的元数据与张量目录、合法维度、对齐、偏移和数据范围；文件不得截断、重叠或包含越界张量。
- safetensors 必须具有有界 JSON 头、唯一字段、受支持 dtype、合法 shape、连续且不重叠的数据范围；不得有空洞、尾随数据或越界偏移。
- 文件扩展名必须与结构验证结果一致，不能通过重命名改变格式判定。
- 下载后先核对可信 SHA-256，再以同一已打开文件的身份信息完成结构验证和原子安装；登记后再次使用时重新计算哈希并验证结构。
- `.pkl`、`.pickle`、`.pt`、`.pth`、`.ckpt` 及其他依赖任意代码反序列化的格式一律拒绝。
- `llama.cpp` 只运行已验证的 GGUF。使用 safetensors 的引擎必须自行提供受管且不执行模型代码的加载路径；没有该路径时明确拒绝运行。

模型下载只接受 HTTPS。Hugging Face 使用解析后的 LFS SHA-256；ModelScope 使用官方仓库文件清单中明确命名为 `Sha256` 的字段。不会把 ETag、文件 ID 或仅长度类似的值推断成 SHA-256。平台清单未提供合法 SHA-256 时要求用户显式提供 `--sha256`，缺失时失败关闭；用户哈希与平台可信哈希同时存在但不一致时同样失败关闭。

## 后果

这会拒绝部分虽能被第三方框架加载、但不能证明不执行代码或结构不严格的模型。拒绝是有意的安全行为，不通过放宽解析器或修改扩展名绕过。新格式必须先增加独立结构验证器、威胁分析、确定性畸形样本测试和本 ADR 的后续决策，才能加入允许列表。

结构验证和 SHA-256 只说明被加载的文件字节满足格式与登记约束，不证明远端节点实际使用了这些字节完成某次 Standard 推理，也不证明其 Token 计数或输出正确。该执行证据边界由信任/传输 ADR 描述，不能从模型验证成功或 Standard receipt 反推。

实现位于 `crates/mindone-engine/src/validation.rs`，下载与登记入口位于 `crates/mindone-cli/src/model.rs`。
