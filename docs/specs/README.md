# 原始规格与当前执行边界

本目录中的两份 `v1.0.0` Markdown 是用户提供的原始输入逐字副本，用于保留需求来源与 SHA-256 证据；它们不是对当前实现能力的独立重新背书，因此不得为了追随实现而改写。

当原始白皮书、CLI 规格与总任务存在冲突时，以总任务、[CLI 合规矩阵](../CLI_COMPLIANCE.md)和已经采纳的 [ADR](../adr/) 所记录的当前边界为准。特别是生产注册账户默认余额为 0；服务器侧受控运营赠额属于任务结算公式之外、经过同事务账本与只追加审计记录的显式外生启动供给，不应被描述为任务自身的通缩结算。详见 [ADR 0002](../adr/0002-accounting-and-reserve.md)。
